## 二、部署方式

### 方式 A：Nginx 静态托管（推荐）

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/degree-english-h5;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # mp3 音频允许范围请求（手机流式播放必需）
    location /audio/ {
        add_header Accept-Ranges bytes;
        add_header Cache-Control "public, max-age=604800";
    }

    # 静态资源缓存
    location ~* \.(css|js)$ {
        add_header Cache-Control "public, max-age=86400";
    }
}
```

### 方式 B：Apache

```apache
DocumentRoot /var/www/degree-english-h5
DirectoryIndex index.html

<Directory "/var/www/degree-english-h5/audio">
    Header set Accept-Ranges bytes
    Header set Cache-Control "public, max-age=604800"
</Directory>
```

### 方式 C：Cloudflare Pages / GitHub Pages / Vercel（免费静态托管）

```bash
# 1. 将 degree-english-h5/ 文件夹上传到 GitHub 仓库
git init
git add .
git commit -m "init: 学位英语点读学习系统"
git remote add origin https://github.com/<user>/<repo>.git
git push -u origin main

# 2. 在平台控制台选择仓库 → 自动部署
# 3. 访问 https://<user>.github.io/<repo>/ 即为网站首页
```

### 方式 D：OSS/COS 对象存储（阿里云/腾讯云）

1. 新建 Bucket，开启「静态网站托管」
2. 将 `degree-english-h5/` 内所有文件上传，保持目录结构
3. 将首页文档设置为 `index.html`
4. 开启 CDN 加速（音频文件较多，CDN 可显著降低加载时间）

## 三、路径规范

所有页面均使用**相对路径**，无绝对路径依赖：

| 资源 | 路径 |
|------|------|
| 全局样式 | `css/style.css` |
| 全局脚本 | `js/app.js` |
| 音频文件 | `audio/w_001.mp3` 等 |
| 页面跳转 | `vocabulary.html`（相对） |

**不可**将文件夹更名或拆散上传，否则相对路径会失效。

## 四、MIME 类型配置

上线前确保服务器对以下类型返回正确的 `Content-Type`：

| 扩展名 | Content-Type |
|--------|-------------|
| `.html` | `text/html; charset=utf-8` |
| `.mp3`  | `audio/mpeg` |
| `.css`  | `text/css` |
| `.js`   | `application/javascript` |

## 六、性能优化建议

1. **音频 CDN**：661个mp3（14MB），建议通过 CDN 加速，减少首次加载延迟
2. **Gzip 压缩**：对 HTML/CSS/JS 开启 gzip（音频文件已是二进制，无需再压缩）
3. **懒加载**：音频文件按需请求（本系统已实现，点击时才加载 mp3）
4. **HTTPS**：部署时强制 HTTPS，否则部分浏览器会阻止音频请求
